1. Introduction

This report presents two integrated machine learning systems using South African datasets. Component A combines ensemble learning (Random Forest, XGBoost) with Q-learning reinforcement learning for traffic accident severity prediction and adaptive safety interventions. Component B implements a transformer-based NLP pipeline for sentiment classification of Parliamentary Hansard Transcripts, combined with a Retrieval-Augmented Generation (RAG) system for policy-focused question answering.

2. Methodology

2.1 Component A: Hybrid RL and Ensemble Learning

Data Preprocessing: Missing values were imputed using median (numeric) and mode (categorical). Categorical variables were label-encoded. Outliers were treated using IQR-based clipping.

Ensemble Learning: Random Forest (200 trees, max_depth=10) and XGBoost (200 estimators, learning_rate=0.1) were trained on an 80/20 stratified split and evaluated using Accuracy, Precision, Recall, and F1-score.

Reinforcement Learning: An MDP with 12 states (4 road conditions x 3 frequency levels) and 4 actions was defined. A Q-learning agent trained over 2000 episodes with decaying epsilon-greedy exploration (1.0 to 0.01).

Integration: Ensemble predictions identify high-risk scenarios; the RL policy recommends optimal interventions per state.

2.2 Component B: LLM and RAG System

Preprocessing: Text was cleaned and tokenized using BPE. Embeddings were generated using TF-IDF (384 dimensions).

Sentiment Classification: A Gradient Boosting classifier was trained on embeddings for 3-class sentiment classification (positive/negative/neutral).

RAG Pipeline: FAISS vector index retrieves top-5 semantically similar documents per query. Answers are generated grounded in retrieved context.

4. Discussion and Ethical Considerations

Bias: Accident data suffers from rural under-reporting, creating geographic bias. Hansard transcripts reflect only parliamentary voices in primarily English, excluding other official languages.

Transparency: Ensemble models provide feature importance but lack local explanations. The RAG pipeline improves transparency by providing source documents alongside answers.

Risks: False confidence in "low risk" predictions could reduce safety vigilance. LLM-generated summaries may misrepresent political positions.

Mitigation: Human-in-the-loop review, confidence thresholds, content filtering, prompt engineering, and regular bias audits across demographics and geography.

South African Context: POPIA requires transparent, contestable automated decisions. With 11 official languages, English-only models exclude significant populations. Models must not perpetuate apartheid-era resource allocation patterns. The NDP 2030 emphasises inclusive technology serving all communities equitably.

5. Conclusion and Recommendations

This project demonstrated the integration of ensemble learning, reinforcement learning, and NLP into functional intelligent systems. Key recommendations:

  • Deploy as decision-support tools with mandatory human oversight
  • Invest in multilingual data collection to reduce language bias
  • Conduct regular bias audits before and after deployment
  • Ensure POPIA compliance in all automated decision processes
  • Implement SHAP explanations for ensemble models to improve stakeholder trust
